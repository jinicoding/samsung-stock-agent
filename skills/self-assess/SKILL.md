# Self-Assessment

You are assessing yourself. Your source code is your body. Read it critically.

## Process

1. **Read your source code** completely (all files under `src/`)
2. **Read memory/active_learnings.md.** Build on what you already know.
3. **Evaluate your analysis capabilities:**
   - Can you calculate technical indicators (MA, RSI, MACD, Bollinger Bands)?
   - Can you analyze supply-demand (foreign/institutional trading patterns)?
   - Can you interpret exchange rate correlation?
   - Can you generate natural language insights from data?
   - Is your Telegram report useful and insightful?
4. **Check data quality:**
   - Is data collection reliable?
   - Are there gaps in historical data?
   - Is error handling adequate?
5. **Check JOURNAL.md.** Don't repeat past mistakes.

## What to look for

- Missing analysis capabilities — what would a securities analyst do that you can't?
- Data quality issues — missing data, stale data, parsing errors
- Report quality — is the Telegram output clear, actionable, insightful?
- Error handling — what happens when KIS API fails? When Naver changes HTML?
- Korean language quality — are outputs natural and professional?
- LLM integration — are you using your LLM brain to interpret data, not just display it?

## Output

Write your findings as a prioritized list:

```
SELF-ASSESSMENT Day [N]:
1. [CRITICAL/HIGH/MEDIUM/LOW] Description of issue
2. ...
```

Then prioritize which ones to tackle this session.
